🔌 REST API ENDPOINTS

1. Add Employee
   POST /api/employee
   Body: {"id": 101, "name": "Alice Johnson"}
   Response: {"status": "success", "message": "Employee added successfully"}

2. Check In
   POST /api/checkin
   Body: {"employeeId": 101}
   Response: {"status": "success", "message": "Check-in successful", 
              "time": "2026-05-19T14:42:12.436725902"}

3. Check Out
   POST /api/checkout
   Body: {"employeeId": 101}
   Response: {"status": "success", "message": "Check-out successful", 
              "time": "2026-05-19T14:42:25.420640265"}

4. Get Employee Attendance
   GET /api/attendance?employeeId=101
   Response: {"status": "success", "data": [{...attendance records...}]}

5. Get Attendance Range
   GET /api/attendance/range?from=2026-05-19T00:00&to=2026-05-19T23:59
   Response: {"status": "success", "data": [{...all records in range...}]}

📞 TROUBLESHOOTING

Issue: Port 8080 already in use
Solution: Kill the process: pkill -f java-jar || Change port in application.properties

Issue: Database file not found
Solution: Automatic - Database creates on first run at ~/attendance.db

Issue: Check out without check in
Solution: System shows error - Check in first, then check out

Issue: API returns error
Solution: Check JSON format, ensure all required fields are present

Issue: Frontend not loading
Solution: Verify server is running, check http://localhost:8080/index.html
