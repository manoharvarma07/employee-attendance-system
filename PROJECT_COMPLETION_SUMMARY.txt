3️⃣  DATABASE (SQLite)
   
   Location: ~/.attendance.db
   
   Tables:
   ┌─ employees
   │  ├── id (INTEGER PRIMARY KEY)
   │  └── name (TEXT)
   │
   └─ attendance
      ├── id (INTEGER PRIMARY KEY AUTOINCREMENT)
      ├── employee_id (INTEGER FOREIGN KEY)
      ├── check_in (TEXT - ISO DateTime)
      └── check_out (TEXT - ISO DateTime, nullable)

4️⃣  REST API (5 Endpoints)
   
   POST   /api/employee
          Add new employee
          Request:  {"id": 5, "name": "John Doe"}
          Response: {"status": "success", "message": "Employee added successfully"}
   
   POST   /api/checkin
          Check in employee
          Request:  {"employeeId": 5}
          Response: {"status": "success", "message": "Check-in successful", "time": "..."}
   
   POST   /api/checkout
          Check out employee
          Request:  {"employeeId": 5}
          Response: {"status": "success", "message": "Check-out successful", "time": "..."}
   
   GET    /api/attendance?employeeId=5
          Get employee's attendance history
          Response: {"status": "success", "data": [...attendance records...]}
   
   GET    /api/attendance/range?from=2026-05-27T00:00&to=2026-05-28T00:00
          Get attendance for date range
          Response: {"status": "success", "data": [...attendance records...]}

🚀 GETTING STARTED

1. Open your browser:
   → http://localhost:8080

2. Navigate the tabs:
   
   Dashboard Tab:
   - View real-time statistics
   - See total employees, today's check-ins, and who's currently working
   - Stats update automatically when you view the tab
   
   Add Employee Tab:
   - Register new employees
   - Enter ID and name
   - Instant feedback on success/error
   
   Check In/Out Tab:
   - Enter employee ID and click "Check In" or "Check Out"
   - View attendance history for any employee
   - See check-in time, check-out time, and duration
   
   Reports Tab:
   - Select date range (from/to)
   - Click "Generate Report"
   - View attendance records with durations in a formatted table
